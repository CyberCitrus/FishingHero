# Fishing Hero 钓鱼勇者

### 一句介绍
来到未完成世界的勇者，正在一边战斗一边……钓鱼？

- 有限状态机实现回合制战斗系统
- 背包系统&模板化道具池
- 简易随机遇敌&模板化敌人池
- 完整游戏驱动流程

### 流程玩法
战斗获取钓鱼机会 -> 通过钓鱼（抽奖）获取战斗道具

### 主要技术点
- 双状态机：外层 `GameState` 控制主流程（探索/进入战斗/菜单等），内层 `BattleState` 控制战斗内回合流程，系统扩展和设计的自由度较高
- 模块通信实现的事件切换：遇敌、战斗结束、菜单关闭等均通过 C# 事件传递，模块独立开发，避免脚本耦合度过高
- 可扩展存档系统：支持玩家位置、队伍状态、剧情状态等的存取（基于 JSON 序列化）
- 数据库管理怪物/道具数据，ScriptableObject形式存储对象资产，使用脚本统一加载查询，支持脱离程序调整数据

### 核心代码结构
- `GameController` : 全局状态机，自由探索/切换到战斗状态
- `battleSystem` : 战斗状态机，可扩展的回合制逻辑
- `PlayerMovement` : 移动、遇敌、存档接口
- `Monster` / `Move` : 数据模型，支持升级与伤害计算
- `MonsterDB` : ScriptableObject 数据库，提供怪物配置

### Bonus
此项目中使用的FSM并不规范，是针对回合制形式使用的enum+switch简易形式，适用于少量状态的迅速构建&减少代码量
对于较为规范的、复用性高的状态机（状态管理器&状态接口&状态类），可以参考root下FSM中的脚本
